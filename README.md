**Claude-powered mock interview coach + profile optimizer + pipeline tracker** for AI engineering contract platforms (Mercor, Outlier, Mindrift, Alignerr).

> One clean localStorage-powered Next.js app. Clone → add one env var → run. Perfect study tool and portfolio piece.

## Why this exists

Self-taught devs targeting high-tier AI contract work need two things:
1. Realistic practice for the ~20-min AI-avatar interviews (Mercor) and timed assessments (Outlier)
2. A profile that signals *systems depth* instead of shallow feature lists (LLM matchers reward architecture, tradeoffs, evaluation)

This app delivers both in one operator-cockpit interface.

---

## Features

### 1. Interview Brief
- Mercor & Outlier deep dives (format, scoring, pay tiers, strategy)
- Record-day setup checklist + Do/Don't list focused on depth over fluff

### 2. Mock Interview (Claude-powered)
- Pick track (Coding / Full-stack / ML-AI / Generalist) + focus area
- One high-signal question generated by Claude (with offline static fallback)
- Live count-up timer
- Type your answer → "Score my answer" (uses your saved profile for context)
- Circular score ring + strengths + sharpen list + stronger rewrite
- Every scored answer logged to local history

### 3. Question Bank
- 8 curated high-signal questions across 4 categories
- Each shows *what it's testing* and *how to answer with depth*

### 4. Profile Builder
- Raw inputs → Claude rewrites into high-tier headline + summary + bullets
- Framed in systems / architecture / evaluation language
- Live preview + one-click save
- Before/After examples included

### 5. Pipeline Tracker
- Quick-add chips for the big four platforms
- Full CRUD: status dropdowns (Researching → Active/Rejected), notes, delete
- Persisted locally

## Tech Stack & Architecture

- **Next.js 16 (App Router) + React 19 + TypeScript**
- **lucide-react** icons (valid current names only)
- **Hand-rolled CSS design system** (no Tailwind) — dark "operator cockpit" with gold/teal/coral accents, Inter / monospace intent, fully responsive + a11y
- **Server-only Anthropic calls**: All Claude logic lives in `app/api/interview/route.ts`. Client never sees the key.
  - POST `{ action, ...payload }`
  - Actions: `question`, `score`, `profile`
  - Model: `claude-3-5-sonnet-20241022` (easy to swap to Haiku or future models)
  - Response parsing: joins `content[].text` blocks
  - Client strips markdown fences + try/catch with clean inline errors
- **Persistence**: `localStorage` keys `idesk_profile`, `idesk_apps`, `idesk_scores`
  - **Supabase drop-in upgrade path** clearly noted in code comments

**Security note for public repo**: The Anthropic key is read exclusively from `process.env.ANTHROPIC_API_KEY` on the server. Never appears in client bundles.

## Local Setup (one env var)

```bash
git clone https://github.com/Berko207/ai-interview-desk.git
cd ai-interview-desk
cp .env.example .env.local
# Add your Anthropic key to .env.local
npm install
npm run dev
```

Then visit http://localhost:3000

Test the API route (requires key):
```bash
curl -X POST http://localhost:3000/api/interview \
  -H "Content-Type: application/json" \
  -d '{"action":"question","track":"Coding","focus":"Technical reasoning"}'
```
